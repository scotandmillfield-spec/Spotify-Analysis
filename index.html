Method & reading notes

  • Day 1 = first day with streams (release day). Leading zero days are stripped; calendar gaps after release are filled with 0.
  • Week n bundles days (n-1)·7+1 … n·7 since release.
  • Columns are auto-detected: the first date-parseable column is the date, the first numeric column is streams.
  • Re-uploading a single merges by date instead of duplicating — newer values overwrite, new days are appended, color and visibility stay put. Matching ignores case, punctuation and Spotify's -timeline / counter suffixes (e.g. (1)), so repeat downloads land on the same single.
  • Singles differ in age — lines run only as far as their data goes, and the shared days figure marks the only window all loaded singles cover.
  • Your data auto-saves in this browser and reloads on open. It lives per device/browser — use Backup to download a JSON copy and Restore to load it on another device or after clearing browser data.
  • Royalties are a separate set from streaming singles. Melino PDF statements are read in-browser; each track's earnings are summed per half-year. You always confirm the detected figures before they're saved, and re-importing the same half-year overwrites it rather than duplicating.
